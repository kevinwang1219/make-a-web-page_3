Stage 1: Work Session 1

In stage 1 we looked further into HTML,CSS and DOM, as well started learning the structuring of a website. We should look at the structure of a website in boxes. It is easier to break down if we look at the page into separate sections, or rectangles.

HTML

HTML is the language of page, which dictates the syntax and rules. It is the standard markup language used to create webpages. It consists of tags, which are "elements" in the tree.

CSS

Also known as Cascading Style Sheets. It is used to style the webpage. It dictates the way the page will look.

DOM

Also known as Document Object Model. It is known for the tree like structure of the code. It helps reading code more organized and easier to read.

The structure, or DOM, of a webpage is built like a tree. Not literally a tree, but there are branches within branches to help organize the page. HTML elements have open tags, as well as closed tags. Each tag can consist of more elements or text. The structure of the page helps identify how the page is classified, in which it tells the browser what is an image, text, and etc.

Think of it like a family: you and your siblings are branches of your parents. Your parents are branches of your grandparents. If you are/were to have kids, they are going to be branches of you. Code is very similar in that aspect.

Avoid Repeating Yourself

Programmers hate having to repeat themselves. Not only does it make it inefficient, but it would be very time consuming when needing to go back to fix the code. It also helps others make the code more readable. If you're consistent in your code, people who want to fix it can do so accordingly and efficiently.

It is also a good way to avoid errors. By having your code more readable, you're able to see where you made the mistakes. For Example: putting some of my classes under the same name because they are going to use the same CSS code. I combined all my notes for each stage under "Lesson-notes, as oppose to using different classes for each one because they are using the same CSS coding.
